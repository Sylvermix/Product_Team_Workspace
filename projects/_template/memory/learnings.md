## What counts as a learning

- User research finding (even a negative one — "users don't want X")
- Validated or invalidated hypothesis
- Technical constraint discovered during implementation
- Pattern that keeps working across features
- Anti-pattern discovered (something we thought would work but didn't)
- Competitive insight from benchmarking

## When to promote a learning to `context.md`

When a learning is confirmed across multiple observations or has become a stable truth the team no longer debates. Example: if 3 rounds of user testing confirm that "users prefer camera over upload for wardrobe items", that moves from learning → assumption in context.md.
